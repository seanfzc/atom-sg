ATOM-SG Pilot Vault
This is the starter vault for the Week 1 baseline + Week 2-5 MVP. Follow the templates to build problem cards, rubrics, and render passes. Use Obsidian with the recommended plugins for tasks, dashboards, calendars, and Git backup.

## 📋 Project Documentation

### Sub-Agent Model Usage Policy
**⚠️ ALL SUB-AGENTS MUST READ:** `MODEL_USAGE_POLICY.md` defines GLM model selection rules:
- **GLM Flash:** Basic/template tasks (lowest cost)
- **GLM 4.7:** Moderate complexity (standard)
- **GLM 5.1:** Complex reasoning (high-stakes decisions)

Refer to this policy before every task to optimize cost vs quality.

### Communication Log
- `01-Projects/SubAgentComms.md` — cross-agent coordination and PM updates
- `01-Projects/KANBAN.md` — task tracking (T1-T5, C1-C4)

Root structure:
- 00-Templates

- 02-Geometry

Key usage notes:
- Problems are named P5-Problem-<id> or DI-Problem-<id>.
- Rubrics use Rubric-<Geometry|Data|BA|CT>_<subtype>.
- Render outputs Render-<ProblemID>.<pdf|md>.
- Milestones are Milestone-Week1, Milestone-Week2, etc.
- Dashboards in 06-Dashboards provide aggregated views of milestones, risks, and problem bank.
